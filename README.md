# Протокол электронного голосования

Распределённая система: 1 сервер (избирком) + 3 клиента (избиратели) + вкладка злоумышленника.

## Файлы

| Файл | Назначение |
|------|-----------|
| `server.py` | Избирком (сервер) — запуск на ПК организатора |
| `client.py` | Избиратель (клиент) — запуск на ПК каждого избирателя |
| `shared.py` | Криптография (RSA) и сетевые утилиты — нужен на всех ПК |

## Требования

- Python 3.10+ (встроенный `tkinter`, дополнительных пакетов нет)

git clone https://github.com/Andrew05812/PAG_MOCI.git

## Проверка на одном ПК

Откройте **4 терминала** в папке проекта:

### Терминал 1 — Сервер (Избирком)
```bash
python server.py
```
1. Вкладка **"Настройка и регистрация"** → кнопка **"Сгенерировать ключи RSA (1024 бит)"**
2. → кнопка **"Запустить сервер"** (IP `127.0.0.1`, порт `9999`)

### Терминал 2 — Избиратель 1
```bash
python client.py
```
1. Вкладка **"Подключение и регистрация"** → IP `127.0.0.1`, порт `9999` → **Подключиться**
2. ID избирателя = **1** → **Сгенерировать ключи RSA (512 бит)**
3. → **Зарегистрироваться** (автоматическая аутентификация challenge-response)
4. Вкладка **"Голосование"** → выбрать голос (За / Против / Воздержался) → **Подготовить бюллетень** → **Отправить голос**

### Терминал 3 — Избиратель 2
Аналогично, но **ID = 2**

### Терминал 4 — Избиратель 3
Аналогично, но **ID = 3**

### Подсчёт голосов (на сервере)
Вкладка **"Подсчёт и проверка"** → **Подсчитать голоса**

### Проверка подсчёта (на клиенте)
1. Вкладка **"Проверка подсчёта"** → **Получить результаты с сервера**
2. → **Выполнить проверку**
3. Результат: **"ПОДСЧЁТ ПРАВИЛЬНЫЙ"** (проверка Q^e mod n = F и R не делится на 2 и 3)

### Демо атаки злоумышленника (на любом клиенте)
1. Вкладка **"Злоумышленник"** → **Сгенерировать ключи злоумышленника**
2. Целевой ID = 1 → **Попытка голосования**
3. Результат: **"АТАКА ОТРАЖЕНА"** (подпись злоумышленника не совпадает с ключом настоящего избирателя)

## Подключение между разными ПК

### Вариант 1: Один WiFi/роутер (самый простой)

Все 4 ПК подключены к одной сети (один роутер/WiFi):

1. На ПК организатора (сервер) узнайте свой локальный IP:
   ```bash
   ipconfig
   ```
   Найдите строку **IPv4-адрес** — будет вида `192.168.1.5`

2. В server.py при запуске укажите IP `0.0.0.0` (слушать на всех интерфейсах), порт `9999`

3. На каждом клиенте при подключении укажите **IP сервера** (например `192.168.1.5`) вместо `127.0.0.1`, порт `9999`

4. Если Windows Firewall спросит — **разрешить** подключение для Python

### Вариант 2: Разные сети (через интернет)

Если ПК в разных сетях — нужен туннель. Проще всего через **ngrok**:
```bash
# На сервере: сначала запустить server.py, потом
ngrok tcp 9999
```
ngrok выдаст адрес вида `0.tcp.ngrok.io:12345` — его указать на клиентах в поле IP.

### Вариант 3: Демонстрация на одном ПК

Если 4 ПК нет — открыть 4 окна терминала на одном компьютере, везде использовать `127.0.0.1`. Каждый клиент — отдельный процесс со своими ключами, протокол работает корректно.

## Протокол

**Этап 1 — Подготовка:** Центр генерирует RSA (1024 бит), избиратели — RSA (512 бит). Открытый ключ центра передаётся автоматически при подключении. Регистрация через challenge-response аутентификацию.

**Этап 2 — Голосование:** Голос: За=2, Против=3, Воздержался=1. Затенение: `ti = b * qi` (qi — случайное простое ≥ 5). Шифрование: `fi = ti^e mod n` (открытый ключ центра). Аутентификация при голосовании. Центр публикует таблицу fi. Защита от двойного голосования.

**Этап 3 — Подсчёт:** `F = ∏fi mod n`, расшифровка `Q = F^d mod n`, факторизация `Q = 2^r · 3^P · R`.

**Этап 4 — Проверка:** Восстановление `Q = 2^r · 3^P · R`, проверка `Q^e mod n = F`, проверка `R % 2 ≠ 0` и `R % 3 ≠ 0`.
