# Протокол электронного голосования

Распределённая система: 1 сервер (избирком) + 3 клиента (избиратели) + вкладка злоумышленника.

## Файлы

| Файл | Назначение |
|------|-----------|
| `server.py` | Избирком (сервер) — запуск на ПК организатора |
| `client.py` | Избиратель (клиент) — запуск на ПК каждого избирателя |
| `shared.py` | Криптография (RSA) и сетевые утилиты — нужен на всех ПК |

## Требования

- Python 3.10+ (встроенный `tkinter`, дополнительных пакетов нет)

git clone https://github.com/Andrew05812/PAG_MOCI.git

## Проверка на одном ПК

Откройте **4 терминала** в папке проекта:

### Терминал 1 — Сервер (Избирком)
```bash
python server.py
```
1. Вкладка **"Настройка и регистрация"** → кнопка **"Сгенерировать ключи RSA (1024 бит)"**
2. → кнопка **"Запустить сервер"** (IP `127.0.0.1`, порт `9999`)

### Терминал 2 — Избиратель 1
```bash
python client.py
```
1. Вкладка **"Подключение и регистрация"** → IP `127.0.0.1`, порт `9999` → **Подключиться**
2. ID избирателя = **1** → **Сгенерировать ключи RSA (512 бит)**
3. → **Зарегистрироваться** (автоматическая аутентификация challenge-response)
4. → **Получить список избирателей** (публикация списка до голосования)

### На сервере: начать голосование
Вкладка **"Голосование"** → **"Начать голосование"** (бюллетени принимаются только в этот период!)

### Избиратели 1, 2, 3 — голосование
Вкладка **"Голосование"** → выбрать голос (За / Против / Воздержался) → **Подготовить бюллетень** → **Отправить голос**

### На сервере: завершить голосование
Вкладка **"Голосование"** → **"Завершить и опубликовать бюллетени"** (таблица fi публикуется один раз после завершения)

### Подсчёт голосов (на сервере)
Вкладка **"Подсчёт и проверка"** → **Подсчитать голоса**

### Проверка подсчёта (на клиенте)
1. Вкладка **"Проверка подсчёта"** → **Получить результаты с сервера**
2. → **Выполнить проверку**
3. Результат: **"ПОДСЧЁТ ПРАВИЛЬНЫЙ"** (проверка Q^e mod n = F и R не делится на 2 и 3)

### Демо атаки злоумышленника (на любом клиенте)
1. Вкладка **"Злоумышленник"** → **Сгенерировать ключи злоумышленника**
2. Целевой ID = 1 → **Попытка голосования**
3. Результат: **"АТАКА ОТРАЖЕНА"** (подпись злоумышленника не совпадает с ключом настоящего избирателя)

## Подключение между разными ПК

### Вариант 1: Один WiFi/роутер (самый простой)

Все 4 ПК подключены к одной сети (один роутер/WiFi):

1. На ПК организатора (сервер) узнайте свой локальный IP:
   ```bash
   ipconfig
   ```
   Найдите строку **IPv4-адрес** — будет вида `192.168.1.5`

2. В server.py при запуске укажите IP `0.0.0.0` (слушать на всех интерфейсах), порт `9999`

3. На каждом клиенте при подключении укажите **IP сервера** (например `192.168.1.5`) вместо `127.0.0.1`, порт `9999`

4. Если Windows Firewall спросит — **разрешить** подключение для Python

### Вариант 2: Разные сети (через интернет)

Если ПК в разных сетях — нужен туннель. Проще всего через **ngrok**:
```bash
# На сервере: сначала запустить server.py, потом
ngrok tcp 9999
```
ngrok выдаст адрес вида `0.tcp.ngrok.io:12345` — его указать на клиентах в поле IP.

### Вариант 3: Демонстрация на одном ПК

Если 4 ПК нет — открыть 4 окна терминала на одном компьютере, везде использовать `127.0.0.1`. Каждый клиент — отдельный процесс со своими ключами, протокол работает корректно.

## Протокол

**Этап 1 — Подготовка:** Центр генерирует RSA (1024 бит), избиратели — RSA (512 бит). Открытый ключ центра передаётся автоматически при подключении. Регистрация через challenge-response аутентификацию. До голосования публикуется список зарегистрированных избирателей.

**Этап 2 — Голосование:** Голос: За=2, Против=3, Воздержался=1. Затенение: `ti = b * qi` (qi — случайное простое ≥ 5). Шифрование: `fi = ti^e mod n` (открытый ключ центра). Аутентификация при голосовании. Бюллетени принимаются **только в период голосования** (организатор нажимает "Начать" / "Завершить"). После завершения — **одноразовая** публикация таблицы fi. Защита от двойного голосования.

**Этап 3 — Подсчёт:** `F = ∏fi mod n`, расшифровка `Q = F^d mod n`, факторизация `Q = 2^r · 3^P · R`.

**Этап 4 — Проверка:** Восстановление `Q = 2^r · 3^P · R`, проверка `Q^e mod n = F`, проверка `R % 2 ≠ 0` и `R % 3 ≠ 0`.

## Где и как реализованы цифровая подпись, шифрование и аутентификация

### Шифрование RSA (shared.py → `rsa_encrypt`, `rsa_decrypt`)

**Используется два раза:**

1. **Шифрование бюллетеня** — избиратель шифрует затенённый голос открытым ключом **центра**:
   - `fi = ti^e_center mod n_center` (`client.py:402`, этап голосования)
   - Расшифровать может только центр своим секретным ключом `d_center`
   - Благодаря гомоморфности RSA: `∏fi = (∏ti)^e mod n`, что позволяет считать голоса без расшифровки каждого бюллетеня

2. **Создание цифровой подписи** — избиратель шифрует challenge своим **секретным** ключом:
   - `signature = challenge^d_voter mod n_voter` (`client.py:300` при регистрации, `client.py:314` при голосовании)
   - Это и есть RSA-подпись: тот, кто знает `d_voter`, может подписать; любой с `e_voter` может проверить

### Цифровая подпись (server.py → `_on_signature`, `_on_vote_response`)

**Схема:** избиратель подписывает случайный challenge от центра своим секретным ключом. Центр проверяет подпись открытым ключом избирателя.

**При регистрации** (`server.py:272-309`):
1. Центр генерирует случайный `challenge` (128 бит) и отправляет избирателю
2. Избиратель вычисляет `signature = challenge^d_voter mod n_voter` — это **цифровая подпись** challenge
3. Центр проверяет: `signature^e_voter mod n_voter == challenge` — если равно, подпись верна

**При голосовании** (`server.py:355-416`) — та же схема повторяется:
1. Центр отправляет новый `challenge`
2. Избиратель подписывает его + отправляет зашифрованный бюллетень `fi`
3. Центр проверяет подпись и только тогда принимает бюллетень

### Аутентификация (challenge-response)

**Реализована** через описанную выше цифровую подпись:

- **Кто ты?** — избиратель предъявляет свой открытый ключ при регистрации
- **Докажи!** — центр отправляет случайный challenge, избиратель подписывает его своим секретным ключом
- **Проверка** — центр убеждается, что `signature^e mod n == challenge`

**Защита от злоумышленника:** злоумышленник не знает `d_voter` настоящего избирателя, поэтому не может создать правильную подпись. Если он подпишет challenge своим ключом, проверка `signature^e_voter mod n_voter != challenge` — подпись не совпадёт (`server.py:292`, `server.py:384`).

### Генерация простых чисел (shared.py → `miller_rabin`, `generate_prime`)

Тест Миллера-Рабина с 20 раундами — вероятность ошибки < 10^(-12). Используется для генерации p и q в RSA-ключах (как центра, так и избирателей), а также для генерации случайного простого `qi ≥ 5` при затенении бюллетеня.

### Ограничение по времени голосования

Организатор управляет фазами: **"Начать голосование"** и **"Завершить и опубликовать"** (`server.py`):
- До начала: бюллетени не принимаются (ответ: "Голосование ещё не началось!")
- После завершения: бюллетени не принимаются (ответ: "Голосование уже завершено! Бюллетень принята вне срока!")
- Регистрация также блокируется после начала голосования

### Публикация данных

- **До голосования:** список зарегистрированных избирателей (ID + открытые ключи) — кнопка "Получить список избирателей" на клиенте
- **После голосования:** таблица зашифрованных бюллетеней fi публикуется **один раз** при нажатии "Завершить и опубликовать" — никаких публикаций после каждого голоса
